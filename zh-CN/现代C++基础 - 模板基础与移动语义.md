# 重载决议

- 因此事实上我们有两个候选：
- template<typename T> void Func(T);（记为 F1）
- template<typename T> void Func(T*);（记为 F2）
- 当我们使用 int* 时，F2 优于 F1；当我们使用 nullptr 时，F1 优于 F2。
- 因此存在内在的匹配顺序，那就是重载决议。

# 重载决议

- 因此最终调用哪个函数，由以下流程决定：
1. 进行名字查找，找出所有可能的函数，形成重载集合。
- 这一步中 ADL 会起作用，但我们不展开讲。
2. 根据原型丢弃不合法的函数，形成可行候选。
- 例如不可推导的模板；
- 例如 Func(int, double) 不能由 Func(1) 调用，因为参数个数不对。
3. 进行重载决议，找出最佳候选。若没有最佳候选，则编译错误。
4. 检查该候选是否能够通过编译。
- 例如若为 =delete，则编译错误（是的，它不会在步骤 2 被排除）；
- 例如函数体中有不满足的 static_assert。

# 重载决议

- 简而言之，重载决议试图根据参数找出 “最精确” 的匹配。顺序是：
1. 完全匹配或仅需最少调整（例如 decay、添加 cv- 限定）。
2. 提升匹配，例如 short->int、float->double。
3. 标准转换匹配（预定义的转换），例如 int->short。
- 对一条转换序列（至多 s-u-s），更短者优先。
4. 用户定义转换匹配。

# 重载决议

- 若仍有多于一个候选，还会应用更多规则：
1. 更特化者优先，包括考虑值类别；
2. 非模板优先于模板；
3. 对指针，转换顺序为：派生类到基类 > void* > bool。
4. 对 initializer_list，在使用花括号初始化时，它优先于其他重载。
- 这就是为什么 std::vector<int>(5, 1) ≠ std::vector<int>{5, 1}。
5. 函数对象优先于 surrogate function（即需要转换才能成为可调用函数对象的那种）。
- ……
- 这非常复杂，我们不再深入；若你有兴趣，参见 [over.match]。

# 重载决议

- 于是我们现在知道原因：

- int*：两者都是精确匹配，但 F2 比 F1 更特化。
- nullptr：F1 精确匹配（T 为 nullptr_t），而 F2 不是。
- 练习：若我们再加 void Func(int*); 作为 F3 呢？
- int*：F3 > F2 > F1，因为在都精确匹配时非模板优先。
- nullptr：F1 精确匹配，F3 需要转换，因此 F1 > F3。若只想用模板，需要显式写 Func<…>()（这样非模板会因语法错误而不进入候选）。

# consteval if

- 存在一种特殊情况——在编译期求值时做一件事，在运行时做另一件事。
